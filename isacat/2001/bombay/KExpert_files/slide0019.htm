IF the light is green THEN you must go
=> you must go = true
IF you must go THEN apply the accelerator
=> apply the accelerator = true
GOAL !
Inference Engine
ØOutput : apply the accelerator = true
Input : the light is green = true
In essence, all the inference engine does is resolve the known facts against its knowledge base. There are two parts to this process - matching the facts with corresponding rules and applying the rules to ascertain the state of more facts.

To introduce a bit of production system terminology, when the whole of the left hand side of a rule has been determined, giving us a result for the right hand side,  the rule is said to have fired.
The system will need to keep track of which rules have fired and the facts which it knows are true or false. The part of the system that holds this information, that is the current state of the problem-solving process is known as the working memory. No, it isnt a very original name.
