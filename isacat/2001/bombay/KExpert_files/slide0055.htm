This is how I test to see if I’ve found the memo I’m looking for. I decided to label the Kex memos with the word Kex and the name of the knowledge base. This code snippet does a bit of pattern matching to see if the memo starts this way. I’ve delimited the name in the memo with inverted commas, so I wrap the name here with these before doing the test.

The string class has a regionMatches method which I can use for the test, and as I’m only interest in the first line, I pull out the first 20 characters of the record byte array and make a string out of these called head.
If the start of the memo matches Kex, then it checks for a match with the name.

When the method finds the record I’m looking for, in other words the memo containing the knowledge base I’ve specied, it returns it.
This will give the system a load of text in XML format and now it has to make sense of it.
