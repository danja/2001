If we look at the attributes of the fact class, we have a series of variables of the same type. To me this is crying out for a data structure. The attributes and values come in pairs, and so this suggests a Hashtable. There is a minor problem with this, in that boolean types are primitives and the Hashtable class can only hold objects. Its easy enough to work around this though.
A very important question we have to ask now is - does the CLDC include a HashTable class?
