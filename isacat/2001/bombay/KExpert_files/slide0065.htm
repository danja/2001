public class KParser{
private XDocumentHandler handler;
private byte[] source;  
private int ch_count;
private static final char END = (char)-1;
public void setSource(byte[] s) {
source = s;
}
   
public void setDocumentHandler(XDocumentHandler xdh) {
handler = xdh;
}
 
public void parse(){
ch_count = 0;
readDoc();
}
Here is the start of the parser code.
 We have a document handler to handle the parse events, a byte array that will contain the XML data we wish to parse, a counter to keep track of where we are in the text and a constant that will be used to tell when we’ve reached the end of the data.
The first two methods here just point the parser at the source data and tell it which which document handler to use.  The third method here will be called when we wish to parse the the text. The internals of this parsing are looked after by a method called readDoc, which itself will call various helper methods.
