Now we have the directory walker itself. When supplied with a Java File object, it first checks to see if it is a directory - directories can be represented as file objects as well as files.
If the file is a directory, we call the appropriate handler method. Next we use File’s list() method to get a list of the files contained in that directory. Again, this is much like the DOS dir command. Here the list is sorted. Now the interesting bit - if the directory isn’t empty, the method will step through each of the items found on the list, which may be files or directories, and call this same method. This kind of recursion is an exceptionally good way of moving around trees of data. The stop condition for the recursion is simple enough - it stops when it has run out of files and directories.
If the directory didn’t contain any files or subdirectories, we call the handler to deal with the empty directory. As it happens, in this application this will do nothing.
Finally, if the File object wasn’t a directory, then it must have been a file, so we call the fileElement handler.
