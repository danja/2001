We’ve now looked at all the important classes of the AutoHelp application bar one - the one which converts the in-memory DOM tree into an XML file. The code in this application uses core Java classes and the Java API for XML Processing (JAXP), also available from Sun.  The easiest way of getting a file out is to use one of the classes from the crimson package, developed by the Apache group. This package is included in the JAXP release, though it may be worth remembering that it isn’t strictly part of the JAXP standard Java extension package.
The method used to write XML to file is found in the OutputDOM class of the AutoHelp application. It couldn’t be easier. First we cast the DOM document in question into a crimson XmlDocument, create an file object, create an output stream from this object and then just write it.
