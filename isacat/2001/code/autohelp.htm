OVERVIEW

This is a utility to automatically create the four XML index files needed by JavaHelp. If your HTML documents are in a filesystem like this :

book\contents.html
book\Chapter1\Page1.html
book\Chapter1\Page2.html
...

you can create JavaHelp views like this :

The tool uses the file and directory names as entry titles, and the directory structure as the contents tree structure. All you need to supply is the root directory name where the HTML files are stored.

There's a little bit more info about this in the form of presentation materials, though your best bet is to get the download and try it. Note that the presentation materials were prepared for v 1.01, so there may be minor differences between these and the latest release. The latest version of the code is strongly recommended - no known bugs, unlike v 1.01.

Use :
Where the title for the HelpSet will be Book and the directory tree containing the html files starts at ./book :

java -classpath .;%classpath%;jaxp.jar;crimson.jar;autohelp.jar com.isacat.autohelp.AutoHelp "Book" book

(see included batch files for more examples)

<?xml version="1.0" encoding="UTF-8"?>

<toc version="1.0">
  	<tocitem text="Appendices">
    			<tocitem text="Appendix1">
      				<tocitem text="Page1" target="Page1" />
      				<tocitem text="Page2" target="Page2" />
    			</tocitem>
    			<tocitem text="Appendix2">
      				<tocitem text="Page1" target="Appendix2Page1" />
      				<tocitem text="Page2" target="Appendix2Page2" />
   			 </tocitem>
 		 </tocitem>
  	 <tocitem text="Chapter1">
    			<tocitem text="Page1" target="Chapter1Page1" />
    			<tocitem text="Page2" target="Chapter1Page2" />
  	 </tocitem>
  	 <tocitem text="Chapter2">
    			<tocitem text="Page1" target="Chapter2Page1" />
    			<tocitem text="Page2" target="Chapter2Page2" />
 		 </tocitem>
    <tocitem text="Contents" target="Contents" />
</toc>

Further information - see the JavaHelp documentation, and any material on Java/XML.

There's a JavaHelp mailing list at Sun - Binky has all the answers. There's also an O'Reilly book "Creating Effective JavaHelp" by Kevin Lewis which I've seen recommended but haven't looked at myself.
