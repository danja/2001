Information Overload Unit Search Engine

One of the classes in the IOU is called XMLProperties, that might find a lot of use elsewhere, most probably for providing config files.

Here's the source : XMLProperties.java and the DTD it uses : jproperties.dtd

The Java Properties class is perfect for config information, I just wasn't happy with the text file format, so I came up with this.

Given the location of an XML file containing sets of property information it will return either a HashMap of all the property sets in the XML or a single named one.

The XML will look something like this : 

<?xml version="1.0"?>
 <!DOCTYPE properties SYSTEM "properties.dtd">
   
   <properties> 
       <propertylist name="danny"> 
           <property name="website">www.isacat.net</property> 
           <property name="email">danny@panlanke.net</property> 
       </propertylist>
   ... 
   </properties>

You can have many propertylists, I did this so I could have a separate set of Properties for different bits of my app, all specified in the one XML file.

To get the properties you need to do something like this :

XMLProperties props = new XMLProperties();
props.load(addresses.xml);
Properties danprop = props.getProperties("danny");

then you can get individual values like :

System.out.println(danprop.getProperty("email"));

(should) output :

danny@panlanka.net
