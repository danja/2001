Creating XML for JavaHelp

We have just seen how we may use DOM to build structured XML, which may have seemed merely an exercise, however interesting. We will now briefly a look at a real-world system that uses structured XML, and coincidentally we will again get the structure from the filesystem. The system in question is JavaHelp. As the name suggests, this was primarily designed with 'help' facilities in mind, but the system is in no way tied to 'help' information, and can be used as a standalone presentation tool for any set of documents that are in HTML format. 

We will look at :

·          How the JavaHelp system uses XML data

·          An overview of a tool based on the same principles as DirToXML for generating this data

·          How to use this to to generate indexes for a set of HTML files to create custom electronic books viewable with the JavaHelp system

The code for this application runs into many pages, so will not be described here in any detail. The source may be downloaded from http://www.wrox.com or http://www.isacat.net.

The application can be used to automatically catalog virtually any tree of HTML documents, the catalog data being output as XML files. JavaHelp can use this catalog to present the tree of documents in a hierarchical 'Contents' form or as a 'Index' list.

The structured information is a set of HTML files arranged into directories, each file being one page to be presented. The presentation tool is part of the JavaHelp system, and here we will be using one particularly straightforward configuration. As we will see shortly, the XML Indexes provide the presentation tool with navigation information for viewing the structured information.
